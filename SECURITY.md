# Security Policy

## Scope

This repository is an automation and data-processing project. Please report security issues responsibly and avoid publishing exploit details in public issues.

## Reporting

If you discover a security issue, contact the repository owner privately through GitHub security reporting or another private channel before opening a public issue.

## What To Include

- affected file or feature
- reproduction steps
- impact summary
- any proposed mitigation

## Sensitive Data

Do not include secrets, tokens, private personal data, or unpublished exploit details in public discussions or pull requests.
